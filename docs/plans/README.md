## Соглашения планов

### Идентификаторы задач
Формат `P<фаза>-<поток>-<номер>`, например `P0-WS1-03`.
Потоки (workstreams) — независимые дорожки внутри фазы, могут вестись параллельно.

### Статусы
- ⚪ `todo` — не начата
- 🟡 `wip` — в работе
- 🟢 `done` — завершена и удовлетворяет Definition of Done
- 🔴 `blocked` — заблокирована (указать чем)

### Definition of Done (общее)
Задача считается `done`, только если:
1. Код соответствует инвариантам системы ([../README.md](../README.md#инварианты-системы-не-нарушать-при-реализации)).
2. Есть проверка работоспособности (запуск/тест/скрин), результат зафиксирован честно.
3. Не нарушены границы: ядро остаётся доменно-независимым, ключи LLM не в клиенте.

### Порядок ведения
- Отмечать статусы прямо в чеклистах плана фазы.
- Открытые вопросы фазы держать в конце её плана; разрешённые — переносить в архитектурные доки.
