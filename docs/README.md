# Praxis — Персональная адаптивная обучающая платформа

> Кодовое имя: **Praxis** (от *practice* — ядро эффективности в task-based practice и петле обратной связи, а не в пассивном контенте).

Мобильное приложение (iOS/Android) для персонализированного изучения **языков** (первичная цель — подготовка к TOEFL/IELTS) и **технологий** (первичная цель — ML/программирование), спроектированное так, чтобы легко расширяться на любые новые предметные области.

Раздел `docs/` — единый источник правды по системе. Вся реализация ведётся на основе этих документов.

## Видение

Не «ещё одна читалка курсов», а **персональный адаптивный тренажёр** с двумя движками эффективности:

1. **SRS (интервальное повторение) на FSRS** — для удержания фактов, слов, определений, концепций.
2. **Task-based practice с AI-скорингом по рубрикам** — для навыков продукции (эссе, речь, код), где AI заменяет репетитора: оценивает по официальным критериям и даёт таргетированную коррекцию.

Универсальность («язык + ML + что угодно позже») достигается **модульной архитектурой над доменно-независимым ядром**: предметные области — это плагины-модули, а не отдельные ветки кода.

---

## Ключевой педагогический вывод (обоснование архитектуры)

Эффективность зависит от **типа навыка**, а не от предмета. Единый примитив — **Activity** — покрывает все типы:

| Тип навыка | Механика | Реализация |
|---|---|---|
| Удержание фактов | Active recall + интервалы | `srs_review` (FSRS) |
| Рецепция (Reading/Listening) | Comprehensible input + timed exam-format | `exam_drill`, `input_comprehension` |
| Продукция (Writing/Speaking/Code) | Продукция → оценка по рубрике → коррекция | `graded_production` |

Ошибки из продукции автоматически питают SRS-колоду через **error-log** — так два движка связаны в единый цикл.

## Две ортогональные оси разделения системы

Обе оси — это **метаданные над одним движком**, а не разные приложения.

- **Ось связности (online/offline)** — свойство `connectivity` каждой Activity. Всё, что ты *делаешь*, работает офлайн; всё, что требует *интеллекта* (LLM), — это отложенный job, резолвящийся при сети.
- **Ось тематики (модули)** — предметные области регистрируются как модули (`languages`, `ml`, …). Ядро не знает, «язык» это или «ML»; оно диспетчеризует по `activity.type`.

Подробности — в [логическом плане](./architecture/02-logical.md).

## Инварианты системы (не нарушать при реализации)

1. **Offline-first.** Локальный SQLite — источник правды для повторений и накопления ответов. Backend — источник правды для AI-контента. Приложение обязано быть полезным без сети.
2. **AI только через backend.** Ключ LLM никогда не в клиенте. Все запросы к LLM идут через FastAPI-gateway (безопасность, кэш, версионирование промптов, контроль расходов).
3. **Доменно-независимое ядро.** `core-engine` не содержит ни одной строки, специфичной для языка или ML. Всё предметное — в модулях.
4. **Единый event log.** Каждый ответ пользователя пишется в `response`. FSRS и адаптивный план читают только этот лог.
5. **Connectivity — флаг, а не форк.** Нет «оффлайн-версии» и «онлайн-версии» приложения. Есть один клиент, знающий доступность каждой Activity.
6. **Рубрики версионируются.** Промпты-оценщики — это данные (реестр с версиями), а не хардкод, чтобы менять критерии без релиза приложения.

## Глоссарий

- **Activity** — атомарная единица обучающего взаимодействия (карточка, эссе-задание, дрилл, задача с кодом). Имеет `module`, `type`, `connectivity`, `payload`.
- **Module (модуль)** — предметная область-плагин (`languages`, `ml`). Декларирует свои типы Activity, рубрики, UI-рендереры, пайплайны импорта.
- **Rubric (рубрика)** — версионированный промпт-оценщик для `graded_production` (например, IELTS Writing band descriptors).
- **Error-log** — поток ошибок пользователя из продукции, автоматически превращаемый в SRS-карточки.
- **Job** — отложенная AI-задача (мост offline→online): создаётся офлайн, резолвится при сети.
- **FSRS** — Free Spaced Repetition Scheduler, современный алгоритм интервального повторения (замена SM-2).
- **core-engine** — доменно-независимое ядро (Activity-движок, FSRS, event log, sync, job-queue).
